# Phase 0 - 초기 세팅 상세 설계

## 목적

프로젝트의 리전, 네이밍, 폴더 구조, 예산 경고를 먼저 확정해서 이후 작업의 흔들림을 줄인다.

## 연결 문서

- [[mwt-execution-plan-v1.2]]
- [[mwt-architecture-v1.3]]
- [[mwt-cost-down-v1.1]]

## 포함 태스크

- P0-001 리전 결정
- P0-002 저장소 구조 생성
- P0-003 공통 네이밍 규칙 정의
- P0-004 AWS 예산 알림 설정

## 상세 설계

### 리전

- 시작 리전: `ap-northeast-2`
- 이유: 한국 사용자 기준 지연시간과 운영 체감이 좋고, 서비스 대상 지역과 리전이 일치해 네트워크 경로가 단순하다

### 환경

- 초기엔 `prod` 하나만 운용
- 로컬 개발은 로컬 환경변수와 mock로 처리

### 네이밍 규칙

`mwt-{domain}-{env}`

### 예산 알람

- 월 예산 상한 설정
- 예산 50%, 80%, 100% 도달 시 경고

## 산출물

- 리전 결정 메모
- 네이밍 규칙 문서
- 폴더 구조
- AWS 예산 알람

## 완료 기준

- 이후 태스크가 동일한 리전/이름 규칙을 공유한다
- 예산 초과를 조기에 감지할 수 있다

## 리스크

- 리전을 나중에 바꾸면 모든 리소스 네이밍과 비용 추정이 흔들린다
